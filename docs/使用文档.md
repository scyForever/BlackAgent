# BlackAgent 使用文档

> 适用范围：本文档面向本仓库当前版本（本地 CLI / 本地 runtime 形态）。BlackAgent 用于公开或已授权黑灰产情报的防御分析，不提供对外 HTTP API，也不支持登录绕过、验证码绕过、私群访问、自动加群或生产处置自动执行。

## 1. 系统定位

BlackAgent 是一个本地运行的黑灰产情报分析 Agent。它把公开/授权来源中的原始文本、链接、账号、联系方式、工具名、黑话等信息，处理成可复核的风险线索和结构化产物。

当前主链路对外只按 5 阶段理解：

```text
输入任务
  -> 安全与任务路由
  -> 历史资产检索
  -> 情报处理流水线
  -> 线索生成与报告
```

展开后是：

```text
Request
  -> Route & Guard
       - PolicyGuard / SourcePolicyGuard
       - routing profile / 预算控制
  -> Asset Retrieval
       - clue pool / entity graph / semantic cache
       - 判断已有资产是否足够
  -> Need Fresh Data?
       ├─ No  -> Clue Promotion -> Report
       └─ Yes -> Collection / Input Records
                  -> Clean / Dedup / Classify / Extract / Normalize / EntityGraph
                  -> CluePromotion / SelectiveRefine / ReviewRoute
                  -> summary、clues、trace、review 信息
```

CLI JSON 里的 `flow_decision_traces` 会保留内部 service 级 trace，便于调试；汇报和使用说明按上面的 5 阶段口径解释，不把内部节点展示成十几步 Agent 链路。

主要入口：

- `python main.py`
- `python scripts/run_agent_cli.py`
- `blackagent`（安装包 entry point）
- Python 内部调用：`src.local_runtime.LocalAgentRuntime`

## 2. 安装与环境准备

### 2.1 Python 版本

项目要求 Python `>=3.11`。

### 2.2 安装项目

```powershell
cd D:\研一\BlackAgent
pip install -e .
pip install -e .[dev]
```

可选能力：

```powershell
# PostgreSQL 后端
pip install -e .[postgres]

# Telegram 授权采集
pip install -e .[telegram]
```

### 2.3 默认安全配置

默认 `config/config.yaml` 使用安全模式：

- `network.enabled=false`：默认不联网。
- `llm.provider=mock`、`llm.enabled=false`、`llm.dry_run=true`：默认不真实调用外部 LLM。
- `enforcement.enabled=false`、`enforcement.dry_run=true`：默认不执行生产封禁/拦截/拉黑。

真实联网或真实 LLM 必须显式开启。

## 4. 常用调查方式

### 4.1 直接输入一条文本

适合验证分类、抽取、线索生成是否正常。

```powershell
python scripts\run_agent_cli.py `
  --query "分析这条内容是否包含接码、群控、引流或账号交易线索" `
  --content-text "群控脚本接码上车，联系 TG:core01，落地 risk.example/path" `
  --show clues `
  --dry-run
```

说明：

- `--content-text` 可重复传入多条。
- 此模式不会联网。
- 输出中的联系方式、URL、TG 等会进入实体抽取和线索生成流程。

### 4.2 使用本地样本文件

输入文件支持 JSON 或 JSONL，每条记录建议包含：

```json
{
  "trace_id": "sample-1",
  "source_name": "manual_sample",
  "source_type": "Manual",
  "legal_basis": "PUBLIC_COMPLIANT_DATA",
  "publish_time": "2026-05-28T01:00:00+08:00",
  "content_text": "待分析文本"
}
```

运行：

```powershell
python scripts\run_agent_cli.py `
  --query "复核样本中的黑灰产风险线索" `
  --fixture-path data\my_fixture.jsonl `
  --show clues `
  --dry-run
```

### 4.3 只使用本地语料 / clue pool

当不传 `--fixture-path`、`--content-text`、`--demo-sample` 且不开启网络时，CLI 会尝试从本地高风险语料加载 query 相关记录。

```powershell
python scripts\run_agent_cli.py `
  --query "找近24小时接码、群控、私域导流相关高质量线索" `
  --routing-profile fast `
  --show clues `
  --dry-run
```

可控制本地语料：

```powershell
python scripts\run_agent_cli.py `
  --query "找接码注册线索" `
  --local-corpus-path data\cleaning_phase_high_risk_corpus.jsonl `
  --local-corpus-limit 100 `
  --show summary `
  --dry-run
```

禁用本地语料自动加载：

```powershell
python scripts\run_agent_cli.py `
  --query "找接码注册线索" `
  --no-local-corpus `
  --show summary `
  --dry-run
```

## 5. 联网采集公开/授权来源

### 5.1 安全边界

联网采集只允许公开或已授权来源。每个 source 必须具备合法来源依据，例如：

- `PUBLIC_COMPLIANT_DATA`
- `AUTHORIZED_PARTNER`
- `INTERNAL_AUTHORIZED_SOURCE`
- `THIRD_PARTY_AUTHORIZED_FEED`

系统会拒绝以下行为：

- URL 内嵌账号密码、token、cookie、session 等凭据。
- `allow_login_bypass=true`
- `allow_captcha_bypass=true`
- `allow_interaction=true`
- `allow_auto_join=true`
- `allow_private_group_access=true`
- `allow_file_download=true`

### 5.2 使用公开 source catalog

```powershell
python scripts\run_agent_cli.py `
  --query "找接码、群控脚本、TG 导流相关的高质量黑灰产线索，输出可复核证据链" `
  --source-config-path config\intel_sources.blackgray.yaml `
  --enable-network `
  --routing-profile balanced `
  --max-sources 2 `
  --max-raw-records 500 `
  --max-candidate-clues 30 `
  --max-llm-refine-clues 5 `
  --max-elapsed-seconds 30 `
  --output data\investigation_result.json `
  --show clues
```

### 5.3 高召回调查

```powershell
python scripts\run_agent_cli.py `
  --query "围绕接码、刷单作弊、账号交易、诈骗引流、众包任务做高召回调查" `
  --source-config-path config\intel_sources.blackgray.yaml `
  --enable-network `
  --routing-profile high_recall `
  --max-sources 5 `
  --max-raw-records 20000 `
  --max-candidate-clues 200 `
  --show clues `
  --output data\blackgray_high_recall.json
```

### 5.4 一键采集黑灰产 raw 数据

```powershell
python scripts\collect_blackgray_all.py --fresh
```

默认输出：

- `data/blackagent_blackgray_all.db`

该脚本会按可用配置依次尝试：

1. 公共网页/搜索页采集。
2. 搜索结果 `source_url` 源站补抓。
3. X recent-search 原生采集（需要 bearer token）。
4. Telegram 授权账号回补（需要 API 凭据）。
5. 可选 clue build 批处理。

如果缺少 X 或 Telegram 凭据，脚本会跳过对应采集器，不影响公开网页链路。

## 6. routing profile 怎么选

配置文件：`config/routing_profiles.yaml`。

| profile | 适用场景 | 主要取舍 |
| --- | --- | --- |
| `fast` | 本地快速复核、低延迟交互 | 默认不做 live collection / query rewrite / record enrich，少量 clue refine |
| `balanced` | 默认调查模式 | 有限 live collection、query rewrite、record enrich、clue refine |
| `high_recall` | 批处理、高召回探索 | 放宽 source 数、候选数、LLM 调用数和耗时预算 |

示例：

```powershell
python scripts\run_agent_cli.py --query "找账号交易线索" --routing-profile fast --show summary --dry-run
python scripts\run_agent_cli.py --query "找账号交易线索" --routing-profile balanced --show summary --dry-run
python scripts\run_agent_cli.py --query "找账号交易线索" --routing-profile high_recall --show summary --dry-run
```

## 7. 真实 LLM 配置

默认不真实调用 LLM。如需接入 OpenAI-compatible provider：

```powershell
$env:BLACKAGENT_LLM_BASE_URL='https://your-provider/v1'
$env:BLACKAGENT_LLM_API_KEY='***'
$env:BLACKAGENT_LLM_MODEL='your-model'
$env:BLACKAGENT_LLM_DRY_RUN='false'
```

复制并调整配置：

```powershell
Copy-Item config\config.real.example.yaml config\config.real.yaml
```

运行真实 LLM smoke：

```powershell
python scripts\smoke_llm_real.py --force-real
python scripts\smoke_llm_real.py --force-real --include-investigation
```

用真实 LLM 跑调查：

```powershell
python scripts\run_agent_cli.py `
  --config config\config.real.yaml `
  --query "复核最近24小时接码和群控相关线索" `
  --fixture-path data\cleaning_phase_high_risk_corpus.jsonl `
  --routing-profile balanced `
  --force-real `
  --show clues
```

注意：当前 `data/eval/latest_llm_value.json` 显示 record enrich 未带来已测质量收益，系统默认倾向只在高价值 clue refine 上使用 LLM，而不是全量 raw 记录都调用 LLM。

## 9. 结果怎么看

### 9.1 CLI summary

常见字段：

| 字段 | 含义 |
| --- | --- |
| `status` | 本次运行是否完成 |
| `mode` | 运行模式，例如 provided_records_pipeline |
| `input_count` | 输入样本数 |
| `high_quality_count` | 高质量线索数 |
| `candidate_count` | 候选线索数 |
| `execution_summary` | 清洗、分类、实体、线索、预算和 trace 汇总 |
| `execution_summary.main_flow_stages` | 代码实际执行的 5 阶段主流程：输入任务、安全与任务路由、历史资产检索、情报处理流水线、线索生成与报告 |
| `collection_runs` | 联网采集时各 source 的抓取结果 |
| `llm_call_traces` | LLM 调用是否成功、是否 fallback |
| `model_route_traces` | 哪些记录或线索走 LLM、哪些只走确定性规则 |
| `flow_decision_traces` | 内部 service 级 trace，解释 5 阶段主链中为什么继续、跳过或回退 |

### 9.2 线索输出

`--show clues` 会展示高质量线索和候选线索。完整字段建议看 JSON：

```powershell
python scripts\run_agent_cli.py --demo-sample --show json --output data\demo_full.json --dry-run
```

重点关注：

- `high_quality_clues`
- `candidate_clues`
- `evidence_chain`
- `risk_type` / `risk_category`
- `entities`
- `quality_score`
- `review_required`

## 12. 生产处置能力边界

`LocalAgentRuntime.execute_enforcement(...)` 目前保留为本地 runtime 能力，默认 dry-run。真正执行封禁、拦截、拉黑等高影响动作必须同时满足：

1. `enforcement.enabled=true`
2. `enforcement.dry_run=false`
3. action 命中 `allowed_actions` 和 `allowed_target_types`
4. `confidence >= min_confidence`
5. 人工审批通过
6. 请求携带生产安全 token
7. 配置真实 connector

本文档不建议在未完成审计、审批和回滚设计前启用真实生产处置。

## 13. 常见问题

### Q1：为什么只传 query 没联网也能有结果？

CLI 会优先查本地高风险语料和 clue pool。如果本地语料不存在或没有命中，会返回证据不足，而不是擅自联网。

### Q2：为什么 `--source-config-path` 不一定会抓？

需要同时传 `--enable-network`，或者在配置里显式开启 `network.enabled=true`。

### Q3：为什么 LLM 没配也能跑？

系统有规则 fallback 和 mock/dry-run 模式。LLM 不可用时，意图解析、规划、线索 refine 会使用 fallback，不影响主流程验证。

### Q4：如何控制成本和时延？

优先使用：

- `--routing-profile fast`
- `--max-sources`
- `--max-raw-records`
- `--max-candidate-clues`
- `--max-llm-refine-clues`
- `--max-elapsed-seconds`
- `--disable-live-collection`

### Q5：如果误报很多怎么办？

优先调整配置，而不是改代码：

- `config/risk_taxonomy.yaml`
- `config/theme_synonyms.yaml`
- `config/slang_dictionary.yaml`
- `config/context_polarity.yaml`
- `config/clue_generation_rules.yaml`

然后复跑清洗、分类/抽取和评测脚本。
