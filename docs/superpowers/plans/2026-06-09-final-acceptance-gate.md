# Final Acceptance Gate Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Freeze the final BlackAgent delivery acceptance scope in one reproducible summary and remove stale acceptance-report ambiguity.

**Architecture:** Add a single acceptance gate script that runs or aggregates existing validation commands, writes `data/final_acceptance_summary.json`, and treats current manual held-out, clue recall, and evidence-pack artifacts as the only authoritative final acceptance sources. Documentation will point to that summary and mark `data/eval_report.json` as stale/non-authoritative.

**Tech Stack:** Python 3.11, pytest, existing JSON/JSONL artifacts, existing evaluation and evidence-pack scripts.

---

## File Structure

- Create `scripts/run_acceptance_gate.py`: subprocess command runner, artifact loader, summary builder, CLI entry point.
- Create `tests/test_acceptance_gate.py`: focused tests for summary aggregation, stale artifact handling, demo path boundaries, and command failure behavior.
- Modify `README.md`: replace old final evaluation references with the unified acceptance gate and demo paths.
- Modify `docs/交付文档.md`: synchronize final metrics and artifact names with `data/final_acceptance_summary.json`.
- Modify `docs/delivery_report.md`: update stale manual held-out and clue recall numbers and point to the final summary.
- Create/update `data/final_acceptance_summary.json`: generated by the new acceptance gate.
- Mark `data/eval_report.json` stale by moving it out of the authoritative final path or writing a stale marker.
