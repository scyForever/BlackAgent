# BlackAgent 交付文档

> 本文档按 `黑灰产情报分析Agent.docx` 的目标说明当前仓库已经交付的工程能力、产物、数字和边界。所有数字来自当前已复跑的本地产物；历史局部 run 不再作为主交付口径。

## 1. 交付结论

当前项目已经落成一条本地可复跑的公开 / 授权黑灰产情报分析流水线：

```text
公开/授权情报采集
  -> 多模态文本物化
  -> 清洗与去重
  -> 风险分类
  -> 实体抽取
  -> 实体图谱与线索质量评估
  -> 人工复核候选与报告
```

当前可声称本地公开 / 授权 manual held-out gold 已就绪；仍不声称线上生产泛化或自动化处置。联网采集、真实 LLM 和生产处置默认关闭或 dry-run。

`data/eval_report.json` 不再作为 hard negative 或最终交付口径；当前文件如存在，按 stale/non-authoritative 处理。最终误报率、人工复核比例和二级分类指标以 `data/manual_heldout_eval_current.json` 和 `data/final_acceptance_summary.json` 为准。

多源证据 smoke 分两层：`data/source_smoke_report.json` 是默认 dry-run catalog smoke，覆盖 IM、论坛/社媒、垂直平台、公众号/文章四个 smoke group，并保留 source class taxonomy 中公众号/文章归属 `social_or_forum` 的边界；`data/source_live_smoke_report.json` 是本地授权 loopback live smoke，`smoke_scope=four_required_source_evidence_groups`，四组都输出 source、URL、raw/hydrated body、capture snapshot URI 和 raw payload URI。验收证据包见 `data/collection_phase_multi_source_evidence_pack_report.json`：300/300 行有 raw snippet、clean text、classification、entities、source URL、crawl/publish time 和 raw payload URI，63 行有 hydrated body 与 snapshot。

LLM 价值证明分两层：`data/eval_llm_ablation.json` 是标准 fixtures 的离线 ablation；`data/eval_llm_hard_ablation.json` 是 point 4 的 hard ablation，固定使用 `tests/evaluation/llm_required_cases.jsonl` 的 50 条 LLM-required 正样本和 `tests/evaluation/context_conflict.jsonl` 的 50 条冲突负样本。当前 hard ablation 数据指纹为 `d39dd67ec673f8ab`，`scenario_consistency.input_counts.classification_records=50`、`hard_negative_records=50`，`high_recall/real_or_configured_fallback` 标注 `provider_status=fallback`、`fallback_reason=real_gateway_dry_run_or_missing_credentials`。该结果只能作为 credentialless/offline 环境下的 value gate 证明，不能冒充真实 provider 证明。

hard ablation 当前未测得 F1 或有效线索增益，`llm_value_gate.should_enable_record_enrich=false`，`reason=llm_added_cost_without_measured_quality_gain`；同步写出的 `data/latest_llm_value_report.json` 将 `record_enrich_policy=conflict_only`，工程亮点是 value gate 在收益不足或无真实凭据时主动限制 LLM 到冲突/高价值复核场景。真实 LLM 价值证明必须配置真实网关凭据并在命令中追加 `--ablation-include-real`；否则报告必须保留 `real_or_configured_fallback` 的 provider status/fallback reason，不能静默转成 mock 结论。
