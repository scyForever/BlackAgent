# 黑灰产情报分析 Agent 需求规格说明书 (PRD)

## 一、 评审摘要

- 

  **项目目标**：构建端到端自动化流水线，将散落的多源异构黑灰产情报转化为结构化的风险线索与受控处置建议 。  

- **编排模式**：**受控探索式编排**（确定性流水线 Backbone + 局部受控自由探索 Sandbox）。

- **一期范围 (MVP)**：聚焦样本级标准化识别流程、基础实体库建设、受控探索沙箱框架及人工复核工作台上线。

- **核心指标**：离线分类准确率 $\ge$ 92%、实体抽取 $F_1$ 值 $\ge$ 88%、分层时延控制（规则路径 P95 < 1 秒，LLM 标准路径 P95 < 8 秒，探索沙箱路径 P95 < 25 秒）。

- **主要风险**：多源采集合规红线、探索式 Agent 幻觉导致的推导泛滥、高对抗话术导致分类失效。

- **核心交付物**：受控探索流水线工程代码、多级动态标签体系、1000 条标杆样本库、结构化实体库、研判复核后台。

## 二、 项目背景与目标

### 2.1 背景分析

当前黑灰产活动（诈骗引流、刷单作弊、帐号买卖、众包任务等）正以高度产业化、团伙化方式运作 。作案情报零散分布在 IM 工具（X、Telegram 群组等）、社媒平台（贴吧、短视频、种草社区）以及垂直站点（众包平台、二手交易平台、技术交流社区）中 。  

这些黑灰产情报具备以下显着特征：

- **体量大**：每日公域及授权渠道产生海量的原始文本、链接和账号动态 。  

- **噪声多**：大量正常聊天、商业推广广告与真实风险事件交织，真假混杂 。  

- **隐蔽强**：黑话变种、谐音字、特殊表情符号、以及故意将引流信息做成海报图（多模态文字）等对抗手段层出不穷 。  

  传统依靠人工搜索研判的方式覆盖不全、响应滞后，难以跟上黑产手法迭代 。大语言模型（LLM）的成熟为自动化处理语义隐晦的表达、结构化提取风险要素带来了全新的工程破局点 。  

### 2.2 系统核心目标

本系统旨在构建一条 **“情报采集 $\rightarrow$ 智能清洗 $\rightarrow$ 意图分类 $\rightarrow$ 实体抽取 $\rightarrow$ 辅助策略生成”** 的端到端自动化分析流水线 。通过将离散、高隐蔽的情报转化为结构化的“作弊剧本”与“候选处置建议”，帮助安全运营团队由“人肉研判”跨越至“智能对抗” 。  

## 三、 业务场景与使用角色

### 3.1 典型使用场景

- **日常情报巡检**：安全运营人员通过看板实时观测各渠道群组新增的黑灰产动态、高危事件及沙箱探索出的新型风险模式。
- **风控策略迭代**：策略工程师定期从系统审查、导出经人工确认的风险实体（如恶意域名、灰产群组联系方式），注入线上风控黑名单。
- **内容安全话术对抗**：内容安全团队捕获探索沙箱推荐的最新微调谐音敏感词及新型黑话候选，更新拦截词库。
- **人机协同反哺闭环**：一线研判人员在工作台对探索式 Agent 生成的“推识别假设”进行纠偏，修正结果秒级同步，反哺 Few-Shot 提示词与词库。

### 3.2 系统用户角色

- **业务安全专家 / 研判人员**：负责消费高价值风险线索，对工作台分流出的疑难、探索沙箱生成的候选假设进行人工审核与标注纠偏。
- **风控策略工程师**：消费结构化实体库与系统候选处置建议，编写、测试并下发风控拦截策略。
- **AI 运维与系统管理员**：负责动态标签体系维护、提示词（Prompt）版本灰度评测、沙箱工具白名单管控及全局 Token/时间预算控制。

## 五、 核心产物定义与判定边界

系统必须明确划分不同层级数据资产的判定界限，避免过度解读单条噪声数据。

### 5.1 风险样本 (Risk Sample)

- **定义**：单条经过系统流水线标准处理、抽取后的细粒度情报记录。

- **判定标准**：任意单条来自授权渠道的数据，只要完成意图分类且至少抽取出 1 个有效风险实体（链接、账号、联系方式、工具名），即判定为风险样本 。  

### 5.2 风险线索 (Risk Clue)

- **定义**：基于强特征关联聚类形成的、具备明确指向性的黑灰产活动实体集 。  

- **判定标准**：必须满足以下任意一种硬性聚合门槛：
  1. 相同的联系方式（如 Telegram ID、微信、QQ）在 48 小时内交叉出现 $\ge$ 3 次。
  2. 相同的引流落地域名或下载链接出现在 $\ge$ 2 个不同的上游渠道中 。  
  3. 相同的黑产推广话术模板在去重后仍高频出现。

### 5.3 作弊剧本 (Cheating Playbook)

- **定义**：多条同类型风险线索深度抽象聚合后，还原出的黑灰产完整作案生命周期链路 。  

- **判定标准**：不能仅基于单条样本生成。必须由多条已证使的风险线索聚合，且大模型推理文本必须同时成功覆盖以下要素中的 2 个以上：
- **作案目标**（受害平台/业务）、**招募渠道**（引流散播点）、**话术特征**、**工具资产**（群控/脚本）、**引流路径**、**账号体系**（接码/买卖）、**结算方式**（跑分/代付）。

### 5.4 对抗方案 (Countermeasure Strategy)

- **定义**：面向业务防线沉淀的落地防御指南。
- **判定标准**：系统仅生成**候选处置建议**，严禁未经审批自动下发生效。每条方案必须关联至明确的风险线索或剧本，具备完整的证据链追踪、预期误伤面评估和建议灰度范围。

## 六、 系统总体架构与技术选型

### 6.1 MVP 一期技术栈收敛

为了快速跑通主流程，一期进行技术栈收敛，降低架构复杂度：

- **计算与编排层**：Python + Celery 任务队列。
- **结构化与向量存储**：**PostgreSQL**。其中，基础关系表存储结构化元数据与实体；利用 **pgvector** 插件直接承载黑话及短文本的话术语义向量索引。
- **非结构化存储**：本地文件系统或通用对象存储（如本地 MinIO 节点），用于落盘原始 payload 及截图。
- **实体关系映射**：一期暂不部署复杂的原生图数据库，通过 PostgreSQL 建立中间关联表（如 `entity_relation_map`）来实现一级的账号与群组关联。

## 七、 Agent 编排机制：受控探索式编排

系统放弃“完全自由探索”的松散 Agent 架构，采用“**确定性流水线 Backbone + 探索式分支 Sandbox**”的混合编排模式。

```
[ 确定性流水线 Backbone ]
 采集 -> 清洗 -> 规则快轨/LLM标准路径 -> 抽取 -> 归一化 -> 入库 -> 触发评估
                                                               |
                                        +----------------------+----------------------+
                                        | (置信度 >= 0.75)                            | (置信度 < 0.75 或 命中未知标签)
                                        v                                             v
                             [ 沉淀至标准结构化实体库 ]                    [ 进入探索式 Agent Sandbox ]
                                                                                      |
                                                                                      v
                                                                           - 相似文本/话术局部检索
                                                                           - 动态黑话变种归纳假设
                                                                           - 跨样本潜在特征聚类
                                                                                      |
                                                                                      v
                                                                           [ 强制安全边界校验 Policy Guard ]
                                                                                      |
                                                                                      v
                                                                           [ 输出：结构化候选对象 ]
                                                                                      |
                                                                                      v
                                                                           [ 人工复核工作台专家裁决 ]
```

### 7.1 确定性主流程 Backbone

负责标准化的工程处理：数据采集、文本清洗、基于规则或高置信度模型的分类、基础要素提取、标准词映射归一化、任务状态机流转。确保 90% 的已知常态数据高效流过。

### 7.2 探索式分支 Sandbox

仅在主流程遇到以下场景时触发：

1. 分类置信度低于硬性阈值（0.75）；
2. 匹配到“未知/待研判”或“冲突冲突”标签；
3. 抽取出从未在词库出现过的异常实体组合、疑似新型变种黑话。

#### 7.2.1 允许自由探索的环节（分析、归纳、假设生成）

- **未知样本分析**：主动检索局部历史库中的相似文本、相似实体，提出“疑似某欺诈话术新型变异”的描述假设。
- **黑话变种归纳**：从上下文中分析出“音符、抖、DY”可能存在相同的语义指向，归纳生成“候选归一化建议”。
- **风险线索聚类**：围绕相同联系方式、高相似域名、相似群组名称，在局部已授权数据集中执行相似度发散关联，组装候选风险线索。
- **策略草拟建议**：草拟“建议监控某高危词组组合”的防御提示，附带证据链。

#### 7.2.2 严禁放开的硬性红线（硬性工程限制）

- **数据采集边界**：严禁 Agent 自主决定去哪里抓、怎么绕限制。采集源必须严格来自系统配置的授权合规清单。
- **生产策略下发**：严禁 Agent 自动执行任何封禁、拦截、拉黑、拉偏、处置账号等线上高影响动作。
- **敏感实体处理**：联系方式、账号 ID、群组 ID 等隐私实体，严禁被 Agent 随意拼接、向外部非授权接口扩散或导出。
- **正式数据变更**：Agent 生成的所有结论，**严禁自动写入**正式词库、正式标签体系或正式策略库。

### 7.3 任务状态机控制流 (Task State Machine)

单条数据在其生命周期内的状态演进由系统严格托管，定义如下：

$$\text{RAW\_COLLECTED} \longrightarrow \text{CLEANED} \longrightarrow \text{PRE\_FILTERED} \longrightarrow \text{CLASSIFIED} \longrightarrow \text{EXTRACTED} \longrightarrow \text{NORMALIZED}$$

- 如果在 `CLASSIFIED` 阶段置信度不足或判定未知，状态机变更为 `REVIEW_REQUIRED` 并强行挂起，数据流引入 `exploration_agent.py`，在其输出 `ExplorationHypothesis` 后，状态机在复核通过前保持挂起，直至人工复核（`APPROVED` / `MISREPORT`）后，方可演进至 `ENTITY_STORED` 和 `STRATEGY_GENERATED`。

## 八、 异常处理、误报控制与白名单机制

- **反爬封禁与合规准入边界**：采集端遇到 403（越权/禁止）、429（频控）、验证码拦截、登录访问控制等对抗情况时，**系统重试该源的采集并抛出系统告警，可以配置分布式代理 IP 池进行强行反爬绕过**。
- **大面积恶意文本垃圾轰炸**：智能清洗层的前置信息熵过滤组件计算文本信息熵。若信息熵偏离正常人类自然语言安全基线，则判定为“纯对抗白噪声”，单条记录标记为 `DROPPED`。
- **上下文语义白名单规则**：**核心强约束规则**——如果特定的高危实体（如某个恶意欺诈域名）所处的上下文句法结构属于“曝光、辟谣、警方通报、安全通告、新闻报道、研究分析”等防御反击性语义语境，系统直接将该实体判别为活动黑产，并做威胁情报背景建档。

## 十、 模型评估与验收指标

### 10.1 离线标杆评测数据集定义

验收系统模型层能力，必须统一基于标准的 **1000 条人工高精度标注的离线评测数据集** 进行混淆矩阵测算。样本包含：诈骗引流类 250 条、账号交易类 200 条、工具交易类 150 条、刷单作弊类 150 条、正常业务白噪声 150 条、新型手法/未知类 100 条。

### 10.2 分层时延验收标准 (P95)

- **轻量规则路径 (Fast Track)**：P95 < 1 秒（规则直接命中，跳过 LLM）。
- **LLM 标准路径 (LLM Track)**：P95 < 8 秒（标准文本大模型全流程解析）。
- **探索沙箱路径 (Sandbox Track)**：P95 < 25 秒（包含多轮检索、局部聚类及假设生成的完备沙箱流程）。

## 十一、 人工复核工作台需求

- **工作台排序规则**：复核队列综合`风险等级 (High->Low) $\times$ 模型置信度倒数 $\times$ 传播频条数`进行智能升序，确保高危害、新型黑灰产事件优先得到研判。
- **编辑与结论状态**：普通研判员可在前端界面修改风险类型、二级标签、实体具体值，并勾选“确认风险、误报、暂不确定、需升级专家、需加入词库”等结论状态。
- **数据流回流机制**：
  - 标记为“误报”的样本自动进入**白名单候选池**与**离线负样本评测集**。
  - 标记为“确认风险且包含新黑话”的样本，一键注入**动态词库候选池**与大模型 **Few-shot 示例样本池**。

## 十二、 安全、合规与数据生命周期

- **数据脱敏与权限隔离**：涉及公民个人身份隐私、受害者敏感手机号/银行卡等信息，入库时必须自动启用 SHA-256 加盐单向哈希或对称加密进行脱敏存储（`masking_status="MASKED_SHA256"`）。
- **数据留存生命周期规划**：
  - `原始情报数据 (RawIntelligence)`：设定严格的滚动生命周期，入库留存 90 天后自动触发定时任务彻底物理删除，以降低法律合规风险。
  - `高级专家确认的标注样本库 / Few-Shot 资产库`：作为企业核心数字资产，允许长期永久保留。
