  主要优化建议

  1. 优先补分类泛化：基于人工 held-out 的典型错误，重点修二级标签和普通教程/论坛/工具白噪声误报；答辩前不要只展示小 gold
     的 0.99 F1，要主动说明人工 held-out 边界。当前已补充普通 automation 广告、V2EX bot 讨论、短信/Steam 验证、
     工具教程等边界测试，并将负样本白噪声二级标签从正式风险二级指标中剥离。

  2. 强化真实多源证据：再跑一版授权 live/source smoke，按 IM、论坛/社媒、垂直平台、公众号/文章四类输出来源、URL、
     hydrated 正文、snapshot/raw payload，减少“搜索摘要当证据”的质疑。当前 `source_smoke_report` 和
     `source_live_smoke_report` 已覆盖四个 smoke group，loopback live smoke 明确不声称外部平台访问。

  3. 收紧线索产出：把 clue overgeneration 降下来，优先提升“有跨源/多次出现/关键实体/证据链”的线索，弱线索继续进候选复核
     池，不作为高价值线索汇报。当前弱线索会进入 archived/review pool，gated gold overgeneration ratio=1.0。

  4. 给 LLM 一个小而硬的价值证明：选 50-100 条规则低置信/冲突样本，用真实 LLM 在固定 token 预算下做 ablation；如果仍无增
     益，就把“价值门控主动关闭 LLM”作为工程亮点。当前 hard ablation 使用 50 条 LLM-required 正样本和 50 条冲突负样本；
     无真实凭据时 `provider_status=fallback`，`record_enrich_policy=conflict_only`。
