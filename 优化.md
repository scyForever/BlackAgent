# BlackAgent 优化点落地记录

## 1. 补人工确认版 held-out 集

原问题：当前 held-out 来自本地公开/授权 seed，但标注由规则生成；答辩时不能把它说成真实人工 gold。

本轮落地：

- `scripts/build_heldout_eval.py` 生成的 held-out 每条都带 `human_review` 框架字段：`status / annotator / review_date / final_risk_categories / final_secondary_labels / conflict_handling / typical_error / notes`。
- `scripts/export_manual_heldout_review.py` 可导出人工复核 CSV 任务包和说明文档，默认目标 50 条确认样本。
- `scripts/validate_manual_heldout.py` 只允许 `confirmed / corrected` 且字段完整的行进入 `manual_heldout_public_authorized`，禁止 pending seed 冒充人工 gold。
- 当前已生成 59 条待复核 seed，`data/manual_review/heldout_review_task_report.json` 状态为 `ready_for_human_review`。

复跑命令：

```powershell
python scripts/build_heldout_eval.py --limit 60 --per-category 12 --output tests/evaluation/heldout_classification.jsonl
python scripts/export_manual_heldout_review.py --input tests/evaluation/heldout_classification.jsonl --output data/manual_review/heldout_review_task.csv --readme data/manual_review/README.md --report data/manual_review/heldout_review_task_report.json --limit 60 --min-target 50
python scripts/validate_manual_heldout.py --input tests/evaluation/heldout_classification.jsonl --review-csv data/manual_review/heldout_review_task.csv --output tests/evaluation/manual_heldout_classification.jsonl --report data/manual_heldout_report.json --min-records 50
```

边界：真实人工确认不能由代码伪造。当前 `data/manual_heldout_report.json` 仍应显示 `insufficient_confirmed_records`，直到人工填写 CSV 并回收验证通过。

## 2. 补真实外部授权源 smoke

原问题：需要 IM/TG、论坛/社媒、垂直站点三类授权说明和采集报告，不能只停留在本地 demo。

本轮落地：

- `scripts/generate_source_smoke_report.py` 在 dry-run 下校验公开 source catalog 的三类覆盖、授权说明、合规状态和采集指标字段。
- 同一脚本支持 `--network-enabled`，只对 `config/intel_sources.public.yaml` 里的公开/授权 URL、allowed domain 和合规元数据做显式 live smoke。
- `scripts/run_live_source_smoke.py` 新增本地 loopback 授权 HTTP smoke，覆盖 `im_or_group / social_or_forum / vertical_or_technical` 三类，验证授权头、401 拒绝、过滤数、重复率和高风险候选数。
- 每类 source 输出 `collected_count / filtered_count / duplicate_rate / high_risk_candidate_count / failure_reason / authorization_statement`。

复跑命令：

```powershell
python scripts/generate_source_smoke_report.py --source-config config/intel_sources.public.yaml --output data/source_smoke_report.json
python scripts/generate_source_smoke_report.py --source-config config/intel_sources.public.yaml --network-enabled --max-records 3 --timeout-seconds 10 --output data/source_external_smoke_report.json
python scripts/run_live_source_smoke.py --output data/live_source_smoke_report.json
```

边界：loopback smoke 证明真实 HTTP 授权采集链路；external smoke 只针对公开/授权 source catalog，不绕登录、验证码、私群或 robots/terms 限制。

## 3. 扩真实 OCR 与图片文字困难集

原问题：OCR 路径成立，但生产 OCR 依赖外部 engine；需要 20-50 条公开/授权截图或海报样本，并标注联系方式、链接、黑话、工具名。

本轮落地：

- `scripts/build_ocr_hardset.py` 生成 20 条本地授权 image-text hardset，并为每条写入 `manual_labels.contact / links / slang / tool_names`。
- hardset 使用 `src/ocr` 的 `BitmapGlyphOCREngine` 和 PBM 海报保证可复现，不新增生产 OCR 依赖。
- 产物 `tests/evaluation/ocr_image_text_hardset.jsonl` 可纳入评估；`data/ocr_hardset_report.json` 记录数量、风险分布、OCR 状态和 claim boundary。

复跑命令：

```powershell
python scripts/build_ocr_hardset.py --output tests/evaluation/ocr_image_text_hardset.jsonl --image-dir data/ocr_hardset_images --report data/ocr_hardset_report.json --count 20
python scripts/evaluate_pipeline.py --gold tests/evaluation/ocr_image_text_hardset.jsonl --classification-granularity auto --dataset-name blackagent_ocr_image_text_hardset_v1 --dataset-kind ocr_image_text_hardset --profile fast --output data/eval_ocr_hardset_report.json
```

边界：该 hardset 验证 image-text 合同和 deterministic pixel OCR 路径；生产 OCR 质量仍取决于注入的外部 OCR engine 与真实授权图片样本。

## 4. 继续降低人工复核负载

原问题：ops dashboard 显示同口径复核率较高，还需针对 `CONFLICT_REVIEW / 未细分 / 群控脚本` 等高频复核桶校准，但不能牺牲安全边界。

本轮落地：

- `config/risk_taxonomy.yaml` 增加 `review_auto_clear.conflict_secondary_labels / conflict_min_confidence / conflict_min_margin`，只允许高证据、高置信、非 review-only 的冲突样本自动消解。
- `src/enhancement/text_intelligence.py` 增加 `review_decision_reason` 和冲突安全消解逻辑，保留弱证据、低置信、未细分、review-only 标签的人工复核。
- `scripts/generate_ops_dashboard.py` 增加 `by_review_decision_reason`，能看到复核负载来自 `category_conflict / fast_classifier_review / theme_only_evidence / secondary_label_unresolved / review_only_secondary_label` 哪一类。
- 当前同口径 `data/ops_dashboard_report.json` 显示 baseline 复核率 `0.514`，fresh 复核率 `0.4148`，相对下降 `0.193`。

复跑命令：

```powershell
python scripts/generate_ops_dashboard.py --classification-summary data/classification_extraction_phase_high_risk_summary.json --source-smoke data/source_smoke_report.json --review-records data/cleaning_phase_high_risk_corpus.jsonl --review-limit 393 --output data/ops_dashboard_report.json
```

边界：本轮只减少高置信稳定细分标签的无效复核；`CONFLICT_REVIEW`、未细分、review-only 和防御语境不清的样本仍走人工复核。
